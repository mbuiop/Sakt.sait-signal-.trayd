آموزش کامل آپلود فایل روی GitHub Pages

راهنمای گام به گام فارسی برای انتشار سایت رایگان با دامنه اختصاصی

لینک نهایی سایت شما به این شکل خواهد بود:

https://username.github.io

(به جای username، نام کاربری GitHub خود را وارد کنید)

با دنبال کردن این مراحل ساده، می‌توانید فایل‌های HTML خود را روی GitHub Pages منتشر کنید و یک سایت رایگان با دامنه شخصی داشته باشید.

1

ایجاد ریپازیتوری جدید در GitHub

  1. وارد حساب GitHub خود شوید
  2. روی دکمه + (بالای سمت راست) کلیک کنید
  3. گزینه New repository را انتخاب کنید
  4. در بخش نام ریپازیتوری، حتماً نام را به این فرمت وارد کنید:
    username.github.io

    به جای username، نام کاربری GitHub خود را وارد کنید

  5. گزینه Public را انتخاب کنید
  6. تیک Add a README file را بزنید (اختیاری)
  7. روی دکمه Create repository کلیک کنید
📌 نکته حیاتی:

نام ریپازیتوری حتماً باید به فرمت username.github.io باشد (با حروف کوچک)

صفحه ایجاد ریپازیتوری جدید در GitHub

2

آپلود فایل index.html

  1. در ریپازیتوری جدید، روی دکمه Add file کلیک کنید
  2. از منوی باز شده گزینه Upload files را انتخاب کنید
  3. فایل‌های خود (مخصوصاً index.html) را به کادر Drag & drop بکشید یا روی choose your files کلیک کنید
  4. پس از آپلود فایل‌ها، به پایین صفحه بروید
  5. در بخش Commit changes:
    • عنوانی مانند Add index.html بنویسید
    • گزینه Commit directly to the main branch را انتخاب کنید
  6. روی دکمه Commit changes کلیک کنید
📌 نکته مهم:

فایل اصلی سایت شما حتماً باید index.html نام داشته باشد و در شاخه اصلی (root) ریپازیتوری باشد

صفحه آپلود فایل‌ها در ریپازیتوری GitHub

3

فعال‌سازی GitHub Pages

  1. در ریپازیتوری خود، به تب Settings بروید
  2. در منوی سمت چپ، روی گزینه Pages کلیک کنید
  3. در بخش Build and deployment → Source:
    • منوی کشویی را باز کنید و Deploy from a branch را انتخاب کنید
    • در بخش Branch، شاخه main را انتخاب کنید
    • در بخش فولدر، گزینه /(root) را انتخاب کنید
  4. روی دکمه Save کلیک کنید
⏱ زمان انتظار:

پس از ذخیره تنظیمات، ۱-۲ دقیقه صبر کنید تا سایت شما آماده شود

صفحه تنظیمات GitHub Pages

4

دریافت لینک سایت

  1. در همان بخش Settings > Pages بمانید
  2. پس از ۱-۲ دقیقه، یک باکس سبز رنگ ظاهر می‌شود
  3. لینک سایت شما در این باکس نمایش داده می‌شود:
    https://username.github.io
  4. روی این لینک کلیک کنید یا آن را در مرورگر خود کپی کنید
🌐 لینک نهایی سایت شما:

https://username.github.io

باکس سبز رنگ حاوی لینک سایت

5

رفرش GitHub Pages در صورت مشکل

اگر سایت شما نمایش داده نمی‌شود (خطای 404) یا لینک ظاهر نشد:

  1. به Settings > Pages برگردید
  2. گزینه Source را به None تغییر دهید
  3. روی دکمه Save کلیک کنید
  4. صبر کنید تا ذخیره شود
  5. دوباره Source را به main و /(root) تغییر دهید
  6. دوباره روی دکمه Save کلیک کنید
  7. ۵ دقیقه صبر کنید و صفحه را رفرش کنید (F5)
🔧 عیب‌یابی مشکلات رایج:
  • نام ریپازیتوری حتماً باید username.github.io باشد
  • فایل index.html باید در شاخه اصلی باشد (نه داخل پوشه دیگر)
  • در تنظیمات Pages، branch را روی main و folder را روی /(root) قرار دهید
  • پس از هر تغییر ۱-۲ دقیقه صبر کنید

صفحه تنظیمات برای رفرش GitHub Pages

خلاصه مراحل

  • 1. ساخت ریپازیتوری با نام username.github.io
  • 2. آپلود فایل index.html در شاخه اصلی (root)
  • 3. رفتن به Settings > Pages
  • 4. انتخاب branch: main و folder: /(root)
  • 5. ذخیره تنظیمات و ۲ دقیقه انتظار
  • 6. دریافت لینک از باکس سبز رنگ
